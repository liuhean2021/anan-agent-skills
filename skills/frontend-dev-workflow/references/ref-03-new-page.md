## 4. 新增页面

新页面不需要容忍历史债务，标准应该高于存量页面。

### 4.1 标准流程（TDD + 2步确认）

```
┌──────────────────────────────────────────────────────────────────────────┐
│  ① 需求确认与复用分析（AI 填写模板） → 等用户确认 → 再继续                   │
│  ② 测试与文件改动清单（AI 列出清单） → 等用户确认 → 再动手写测试与代码         │
│  ③ 跑测试 + 视觉验证（测试全绿 + 截图走查）                                │
│  ④ 代码审查                                                              │
└──────────────────────────────────────────────────────────────────────────┘
```

> 新页面的测试先行是经典 TDD：先根据需求写失败测试，再写代码让测试通过。
> 好处：需求理解有偏差时，写测试阶段就能发现，而不是写完整页面后才发现方向错了。

### 4.2 Step ①：整理需求建议与复用分析，等用户确认

⚠️ **禁止跳过此步直接写代码。必须先输出以下合并建议，等用户确认后再继续。**

根据用户描述和项目现状，**用真实内容填写以下模板后展示**（信息不足的项标注 `[需确认]` 并在末尾追问）：

```
### 1. 需求建议
新增页面：[页面名称]
路由：[路由路径]
布局：与 [已有页面] 相同的布局结构

功能：
1. [区块1]：[描述内容和组件]
2. [区块2]：[描述内容和组件]
3. [区块3]：[描述内容和组件]

响应式要求：
- ≥ 768px：[桌面端布局描述]
- < 768px：[移动端布局描述]

交互：
- [按钮/链接1]：[行为描述]
- [按钮/链接2]：[行为描述]

### 2. 复用分析
- 参考模板：[找到项目中布局最相似的已有页面]
- 可复用组件：[列出可复用的公共组件/业务组件]
- 样式方案：使用项目现有变量 / token / 主题，不创建等价重复组件

← 以上是初步建议，请确认或说明需要调整的地方
```

> **常见缺失项**（信息不足时标注 `[需确认]` 并追问）：功能模块范围、数据字段、交互方式（弹窗/跳转）、移动端处理方式、导航入口、有无设计图。

### 4.3 Step ②：列出测试清单与文件改动清单，等确认再动手

⚠️ **在写任何代码（包括测试代码）之前，必须先列出此清单等用户回复。**

基于 Step ① 的结论，**用真实路径和描述填写后展示**：

```
### 1. E2E 测试清单 (TDD)
即将为 [页面名] 编写以下测试用例（使用 data-testid 且 Mock 所有 API）：
1. 页面加载后关键元素可见（[区块1]、[区块2]）
2. [交互流程1] 可走通
3. 异常状态展示正常（空数据/加载失败）
4. 响应式：768px 下无溢出

### 2. 文件改动清单
1. 新建：[页面文件路径]
2. 修改：[API 接口文件]（追加接口声明）
3. 修改：[类型定义文件]（追加类型）
4. 修改：[路由配置文件]（追加路由）

← 确认后我将开始写测试（红灯）并随后生成页面代码（绿灯）
```

**为什么测试先行**：
- 写测试时你会发现需求中的模糊点（如按钮具体点击后的反馈）
- 测试用例就是可执行的验收标准，后续不用额外对照
- 页面写完后跑一次测试就知道是否达标，不用人工逐项检查

### 4.4 跑测试、视觉验证与代码生成

1. **写测试（红灯）**：根据 Step ② 确认的清单编写 Playwright 测试。此时页面未创建，测试应全部失败。
2. **写代码（绿灯）**：根据需求和复用分析生成页面代码。
   - 必须为所有交互元素增加 `data-testid`（与测试用例对应）。
   - 必须复用已有组件和样式 token。
3. **跑测试直到全绿**：修复代码直到测试全部通过，不准为了让测试通过而改测试本身。
4. **视觉验证**：截图并与预期对比（如有设计图），确保 1440px 和 768px 下均表现正常。

### 4.5 新页面 data-testid 规范

```
命名规则：[页面]-[区块]-[元素]

示例：
data-testid="order-detail-edit-btn"
data-testid="order-detail-goods-table"
data-testid="order-detail-total-amount"
```

> data-testid 是框架无关的。无论 Vue / React / Angular / Svelte，写法完全一致。
