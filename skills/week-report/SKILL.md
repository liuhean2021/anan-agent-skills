# 前端工作周报生成

## 默认配置

以下为内置默认值，Step 1 交互时用户可覆盖：

```
DEFAULT_REPOS:
  - /path/to/your/repo 项目名称

DEFAULT_TEMPLATE: assets/week-report-template.xlsx
DEFAULT_OUTPUT:   ~/Desktop
USER_EMAILS:      your@email.com
MODULE_MAP:       见 assets/module_map.json（仅在用户未提供项目名称时 fallback 使用）
```

## Excel 模板结构（固定，不可改变）

```
Row 1 : 标题「前端工作周报」（A1:E1 合并）
Row 2 : 表头 — A=本周工作内容  B=所属模块  C=人员  D=工作重点  E=备注
Row 3起: 数据行，每条工作项占一行
```

列写入规则：
- **A 列**：每行一条工作内容描述
- **B 列**：每行对应所属模块（来自 module_map）
- **C 列**：仅第一行写姓名，其余为空；所有数据行合并为一格
- **D 列**：仅第一行写工作重点（全局总结），其余为空；所有数据行合并为一格
- **E 列**：留空

## Step 2 — 扫描 git 提交

对每个仓库执行：

```bash
git -C <repo> log \
  --all \
  --format="%h|%ad|%ae|%an|%s" \
  --date=short \
  --since="<起始日期>" \
  --until="<截止日期>"
```

过滤规则：
- 仅保留 `%ae`（作者邮箱）在 USER_EMAILS 列表中的提交
- 记录：`仓库名 | 日期 | commit subject`

## Step 3 — 脱敏处理

对原始 commit 数据执行以下过滤，确保最终 Excel 中不包含任何敏感信息：

| 类型 | 原始内容示例 | 处理方式 |
|---|---|---|
| 邮箱地址 | `your@email.com` | 过滤掉，不出现在输出 |
| commit hash | `a1b2c3d4e` | 过滤掉 |
| gitlab/内网 URL | `your-gitlab.com/...` | 过滤掉 |
| 内网文件路径 | `/path/to/internal/...` | 过滤掉 |
| Authored-by 行 | `Authored-by: Your Name ...` | 过滤掉 |
| 其他人员提交 | 非 USER_EMAILS 的提交 | 不纳入统计 |

脱敏后仅保留：仓库名、日期、commit subject 的业务描述部分。

## Step 4 — 归类与 AI 归纳

**模块映射**：
- 优先使用 Step 1 中用户输入的「项目名称」作为所属模块
- 若用户未提供项目名称，则 fallback 读取 `assets/module_map.json` 按关键词匹配

**合并规则**：
- 同一仓库多条 `docs/chore` 类工程配置提交（CLAUDE.md、.gitignore 等）→ 合并为一条
- 不同仓库的同类工程配置提交 → 合并为一条，模块写「各项目」

**AI 归纳要求**：
- 用中文重新描述工作内容，面向业务，去掉技术细节前缀（如 `fix:` `chore:` 等）
- 每条简洁，15 字以内为佳
- 最后生成「工作重点」：对所有条目的一句话总结（不超过 30 字）

## Step 5 — 内容确认（必须）

将归纳后的工作条目以表格形式展示给用户，**必须等待用户确认后才能进入 Step 6 生成 Excel**。

展示格式：

```
以下内容将写入周报，请确认是否正确：

| # | 工作内容 | 所属模块 |
|---|---------|---------|
| 1 | xxx     | 模块A    |
| 2 | xxx     | 模块B    |
...

工作重点：<一句话总结>
姓名：<姓名>

请确认：
  → 输入 "确认" / "y" / 回车 → 进入 Step 6 生成 Excel
  → 输入修改意见（如 "第2条改为xxx" / "删掉第3条" / "工作重点改为xxx"）→ 按意见修改后重新展示本步骤
  → 输入 "取消" → 终止流程
```

**关键规则**：未经用户确认，不得执行 Step 6。

## Step 6 — 生成 Excel

**输出文件名**：自动计算当前日期是当月第几周

```
周次计算规则：当月1号所在周为第1周，依次递推（以周一为起始）
文件名格式：前端工作周报{M}月第{N}周.xlsx
示例：前端工作周报4月第1周.xlsx

同一周多次生成时自动版本递进，不覆盖已有文件：
  第1次：前端工作周报4月第1周.xlsx
  第2次：前端工作周报4月第1周_v2.xlsx
  第3次：前端工作周报4月第1周_v3.xlsx
```

调用脚本：

```bash
python3 scripts/gen_report.py \
  --template "<模板路径>" \
  --output "<输出路径>" \
  --name "<姓名>" \
  --focus "<工作重点>" \
  --items '<[{"work":"...","module":"..."},...]>'
```

脚本行为：
1. 检查并自动安装 openpyxl（如未安装）
2. 复制模板到新文件（**原模板文件不做任何修改**）
3. 取消数据区所有合并单元格（保留 A1:E1 标题合并）
4. 按条目逐行写入 A/B/C/D/E 列
5. 动态合并 C、D 列（跨所有数据行）
6. 清空模板中多余的旧数据行
7. 保存输出文件

## Step 7 — 输出确认

生成完成后展示：

```
✅ 周报已生成：<输出路径>

本次填入内容：
1. <工作内容> | <模块>
2. <工作内容> | <模块>
...
工作重点：<一句话总结>
姓名：<姓名>
```

提示用户核对内容是否需要调整。

## 约束

- 原模板文件绝对不能被修改，只能复制后操作副本
- Excel 中不得出现任何邮箱、hash、URL、内网路径等原始 git 信息
- 工作内容描述必须是中文业务语言，不直接复制 commit message
- 姓名为必填项，若用户跳过且无默认值则再次提示
